在Vue+Babel+Webpack环境中使用ESLint

在前端工程化思想盛行的工业化时代，前端代码规范在团队开发中显得越来越重要。前端代码规范的主要目的是在团队协作中避免低级bug、生产出风格统一的代码，通过编写代码规范来建立具有一致性、可执行的样式风格。通过使用定义良好的样式规范和健壮的代码审查。最常见的做法就是使用Lint工具和代码风格检测工具，它可以辅助编码规范执行，有效控制代码质量。

常见的代码规范工具有：

  • jslint:JSHint是一个JavaScript的代码质量检查工具，主要用来检查代码质量以及找出一些潜在的代码缺陷。
  • eslint
  • csslint:csslint是一个用来帮你找出CSS代码中问题的工具，它可做基本的语法检查以及使用一套预设的规则来检查代码中的问题，规则是可以扩展的。
  • htmlhint:htmllint 较好地同时支持了格式规则及语义规则,同时也支持自定义规则。
  • scss-lint:使scss代码持续保持简洁、高效和可维护性。

下面主要介绍ESLint：

ESLint是js中目前比较流行的插件化的静态代码检测工具，它是一个以可扩展、每条规则独立、不内置编码风格为理念编写的一个lint工具。主要的特点有：

  • 默认规则包含所有 JSLint、JSHint 中存在的规则，易迁移
  • 规则可配置性高：可设置「警告」、「错误」两个 error 等级，或者直接禁用
  • 包含代码风格检测的规则（可以丢掉 JSCS 了）
  • 支持插件扩展、自定义规则
  • 支持JSX、ES2015语法

ESLint的使用

  1. 安装eslint包
    在项目的根目录运行： 
     
 npm install eslint --save-dev
  2. 安装eslint-loader
    告诉webpack我们在构建时使用eslint 
     
 npm install eslint-loader --save-dev
  3. 在webpack中配置使用eslint-loader 
     
  module: {  
     loaders:[  
         {
         test: /\.js$/, 
         loader: 'babel!eslint', 
         exclude: /node_modules/
         }
     ]
  }
    或者使用preloaders： 
     
 module: {
 preLoaders: [
  {
     test: /\.js$/,
     exclude: /node_modules/,
     loader: 'eslint'
     },
 ]
 }
    此外，为了检测.vue文件中的JS代码，还需添加配置： 
     
 ...
 module:{},
 ...
 vue:{
     loaders:{
         js: 'babel!eslint'
     }
 }
 ...
  4. 制定检测规则文件
    根目录下 
     
 touch .eslintrc
    .eslintrc 
     
 {
     "rules": {
     }
 }
    webpack.config.js 
     
 ...
 entry:{}
 ...
 eslint: {
     configFile: './.eslintrc'
     }
 ...
  5. 如果现在启动app，会看到The keyword ‘import’ is reserved的解析错误。这是因为eslint还不知道通过Babel使用的ES6特性(比如import) ,我们需要通过babel-eslint来检测ES6代码。 
     
 npm install --save-dev babel-eslint
    修改.eslintrc 
     
 {
     "parser": "babel-eslint",
     "rules": {
     }
 }
  6. 添加ESLint规则

     
     …
     “rules”: {
    "eqeqeq": [2, "allow-null"]

     }
     …

    这条规则表示：使用 === 替代 == allow-null允许null和undefined==
    启动app，会看到不规范代码的提示。
  7. 扩展ESLint规则
    我们不想每次都指定这些规则，所幸已经有很多符合最佳实践的规则。其中之一就是Airbnb Style Guide，此外Airbnb还开源了他们自己的ESlint配置。
    安装Airbnb的lint规则 
     
 npm install eslint-config-airbnb eslint-plugin-import --save-dev
    应用Airbnb的lint规则 
     
 {
     parser: "babel-eslint",
       "extends": "airbnb",
     "rules": {
           "eqeqeq": [2, "allow-null"]
     }
 }
  8. 屏蔽规则
    如果我们不想看到no-unused-vars(为使用过的变量定义)的警告，可以： 
     
 {
     parser: "babel-eslint",
       "extends": "airbnb",
     "rules": {
         "no-unused-vars": 0,
           "eqeqeq": [2, "allow-null"]
     }
 }
    这种是全局的配置，如果是只想在某些文件中禁止检测，可以如下修改(通过注释的方式) 
     
 /*eslint-disable no-unused-vars*/
 import Vue from 'vue';
 /*eslint-enable no-unused-vars*/
 import Vue from 'vue';
 ...
  9. 使用pre-commit钩子
    使用了pre-commit钩子后，在git commit时会自动检测代码规范，如果有违反规则代码的情况，则commit失败。
